üîß Review Permission Fix Guide

‚ùå Problem Identified

Issue: You added a permission called "review" to your "manage product" role

Error: The ReviewController is looking for specific permission names like "REVIEW:CREATE", "REVIEW:EDIT", etc.

Result: Authorization denied when trying to create/update reviews

üîç What ReviewController Actually Needs

For Creating Admin Reviews: REVIEW:CREATE or REVIEW:EDIT

For Updating Reviews: REVIEW:EDIT or REVIEW:UPDATE

For Viewing Reviews: REVIEW:VIEW or REVIEW:EDIT

For Approving Reviews: REVIEW:EDIT or REVIEW:APPROVE

For Deleting Reviews: REVIEW:DELETE or REVIEW:EDIT

‚úÖ Solution: Add These Permissions

Step 1: Go to Role Management

Navigate to your admin panel ‚Üí Role Management ‚Üí Edit "Manage Product" role

Step 2: Add These Permissions

REVIEW:CREATE - Create reviews
REVIEW:EDIT - Edit reviews
REVIEW:VIEW - View reviews
REVIEW:APPROVE - Approve reviews
REVIEW:DELETE - Delete reviews
REVIEW:UPDATE - Update reviews

Step 3: Remove the Old Permission

Remove the permission called "review" (without the colon format) and replace it with the specific permissions above.

‚ö†Ô∏è Important: Permission Naming Convention

Correct Format: RESOURCE:ACTION

Examples:

  • ‚Ä¢ PRODUCT:CREATE - Create products
  • ‚Ä¢ PRODUCT:EDIT - Edit products
  • ‚Ä¢ REVIEW:CREATE - Create reviews
  • ‚Ä¢ REVIEW:EDIT - Edit reviews
  • ‚Ä¢ ORDER:VIEW - View orders

Wrong Format: review (no colon, no action)

üîß Quick Fix Script

// Run this in your browser console to add the correct permissions
// Replace 'YOUR_ROLE_ID' with your actual "Manage Product" role ID
const roleId = 'YOUR_ROLE_ID'; // Get this from the role management page
const reviewPermissions = [
'REVIEW:CREATE',
'REVIEW:EDIT',
'REVIEW:VIEW',
'REVIEW:APPROVE',
'REVIEW:DELETE',
'REVIEW:UPDATE'
];
// First, get all available permissions
fetch('/api/permissions')
.then(response => response.json())
.then(permissions => {
const reviewPermissionIds = permissions
.filter(p => reviewPermissions.includes(p.name))
.map(p => p.id);
// Add these permissions to the role
return fetch(`/api/roles/${roleId}/permissions`, {
method: 'POST',
headers: { 'Content-Type': 'application/json' },
body: JSON.stringify(reviewPermissionIds)
});
})
.then(response => response.json())
.then(result => console.log('‚úÖ Review permissions added:', result))
.catch(error => console.error('‚ùå Error:', error));

‚úÖ After Adding Permissions

1. Refresh the page to reload your permissions

2. Try creating/editing reviews again

3. The authorization should now work because your role has the correct permission names

4. Check the browser console - you should no longer see the React error #418
